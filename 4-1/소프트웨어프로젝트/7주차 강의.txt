사물인터넷
M To M : 기계와 기계를 이용하여 사람을 대신할 수 있도록 하자
             기본적인 처리능력을 가진 machine learning : M
=> IOT로 발전(인터넷을 기반으로 모든 사물들을 연결하여 사람, 사물 또는 사물과 사물 간의 정보를 상호 소통하는 지능형 기술 및 서비스
=> 외부에서 정보를 기계적으로 받아들여야 함(센서)=>사람의 개입 없이 외부 센서를 통해 정보를 받아들임
=> 센서로 취득한 데이터를 클라우드에 업로드하여 인공지능이 학습한 내용에 따라 판단
=> 빅데이터로부터 정보를 빼내야 함(AI)

구현 기술 요소
-사물과 주위 환경으로부터 정보를 얻는 센싱 기술
-사물이 인터넷에 연결되도록 지원하는 유무선 통신 및 네트워크 인프라 기술
-각종 서비스 분야와 형태에 적합하게 정보를 가공하고 처리하거나 각종 기술을 융합하는 서비스 인터페이스 기술
-사물인터넷 구성 요소에 대한 해킹, 정보 유출을 방지하기 위한 보안 기술

- 신체 -> 웨어러블, 가전제품 -> 스마트홈,  자동차 -> 스마트카, 의료 -> 스마트헬스, 공장 -> 스마트 팩토리, 도시 -> 스마트시티

사물인터넷 관련 제품 세 가지 단계 : 센서, 무선통신, 서비스

단계
1. Add-on 스마트 기기 : 기존 제품에 IoT기기 추가
2. Stand-alone 스마트 기기 : 단독 제품의 IoT기기
3. 연결형 스마트 기기 : 사람의 개입 없이 스스로 일을 처리해서 실행하기까지 함

빅데이터 : 디지털 환경에서 생성되는 데이터, 그 규모가 방대하고 생성 주기도 짧고 형태도 수치 데이터 뿐만 아니라 문자, 영상 데이터를 포함하는 대규모 데이터
-> 빅데이터 특징 : 규모(데이터의 크기), 다양성(다양한 종류의 데이터를 수용하는 속성), 속도(데이터를 빠르게 처리하고 분석할 수 있는 속성)
-> 소스 -> 수집 -> 저장 -> 처리(맵리듀스 : 분산하고 모아서 처리) -> 분석(AI가 쓰임) -> 표현(사람이 이해할 수 있도록 보여주어야 함)
빅데이터 처리 방식은 맵리듀스로 변환 => 단순한 데이터 구조를 써서 저장하자
생성(내, 외부데이터) -> 수집(크롤링, ETL) -> 저장(응용에 최적화된 db를 쓰자) -> 처리(맵리듀스가 기본) -> 분석(NLP, 기계학습, 직렬화 등) -> 표현(가시화, 획득)
